# <a name="set-up-loads-to-add-imported-files-in-advanced-ediscovery-classic"></a>Advanced eDiscovery(클래식)에서 가져온 파일을 추가하기 위한 로드 설정

> [!NOTE]
> Advanced eDiscovery를 사용하려면 Office 365 E3의 고급 준수 추가 기능이나 조직을 위한 E5 구독이 필요합니다. 이 요금제가 없는 상태에서 Advanced eDiscovery를 사용하려는 경우에는 [Office 365 Enterprise E5 평가판을 등록](https://go.microsoft.com/fwlink/p/?LinkID=698279)할 수 있습니다. 
  
Advanced eDiscovery에서 로드는 사례에 추가된 파일의 새로운 배치입니다. 기본적으로 하나의 부하가 정의되고 가져온 모든 파일이 추가됩니다. 관계 교육을 수행하기 전에 가져온 파일을 로드에 추가해야 합니다. 
  
다음과 같은 시나리오를 고려합니다.
  
- 새 파일은 사례 데이터베이스에 로드된 이전 파일과 비슷하거나 이전 파일 로드가 파일 모음에서 임의로 설정된 것으로 알려져 있습니다. 이 경우 가져온 파일을 현재 파일 로드에 추가합니다.
    
- 새 파일은 이전 파일과 다르거나(예: 다른 원본에서) 이전 로드와 유사하거나 다를 수 있습니다. 이 시나리오에서는 가져온 파일을 새 파일 로드에 추가합니다. Advanced eDiscovery는 이를 롤링 로드 시나리오로 인식하고, 추가 프로세스를 호출하고, 관련성 학습 및 일괄 처리 계산을 잠그고, 새로운 부하가 통합 및 교육될 때까지 이를 잠급니다. 
    
## <a name="adding-imported-files-to-the-current-load"></a>현재 로드에 가져온 파일 추가

Advanced eDiscovery에서 처리하려면 가져온 모든 파일을 로드에 추가해야 합니다. 가져온 파일이 마지막으로 정의된 로드에 추가됩니다. 나중에 추가 파일을 가져오는 경우 로드에도 추가해야 합니다.
  
1. In the **Relevance \> Relevance setup** tab, select **Loads.**
    
    ![관련성 설정 로드 탭](../media/278aac7f-655f-462f-852a-6baa5d818768.png)
  
2. **파일 포함:** 포함할 파일 옵션을 선택합니다. 기본적으로 현재 부하에 파일을 추가하는 것은 "모든 파일" 인구를 기반으로 합니다.
    
    > [!TIP]
    > 사용 가능한 모든 컬링된 파일을 다른 파일에 로드합니다. 사용 가능한 파일의 하위 집합만 로드하려면 먼저 지원에 문의하시기 바랍니다. 하위 집합을 로드하면 관련성 교육에 부정적인 영향을 줄 수 있습니다. 
  
3. Loads **관리에서** 부하를 선택합니다.
    
4. 파일 **추가를 클릭합니다.** 파일이 로드에 추가되고 확인 메시지가 표시됩니다. 
    
5. **확인** 을 클릭합니다.
    
이제 파일을 고급 eDiscovery에서 처리하여 파일을 학습할 수 있습니다.
  
## <a name="editing-a-load-name-within-a-case"></a>사례 내에서 로드 이름 편집

로드 이름을 변경하는 경우 사례에 중요한 이름을 사용하는 것이 좋습니다.
  
1. In the **Relevance \> Relevance setup** tab, select **Loads.**
    
2. Loads **관리** 목록에서 로드를 선택하고 편집 **아이콘을** 클릭합니다. 로드 편집 창이 표시됩니다. 
    
3. 변경 내용을 입력하고 확인을 **클릭합니다.**
    
## <a name="adding-imported-files-to-a-new-load"></a>가져온 파일을 새 로드에 추가

해당 교육을 시작하거나 일괄 계산을 수행한 후 추가 파일 집합을 가져와서 처리해야 할 수 있습니다. 
  
Catch-up 중에 Up 집합을 만들고 태그를 지정하고 분석할 수 있습니다. Advanced eDiscovery는 새 부하의 관련 파일 및 비관용 파일에 대한 평가를 이전 로드의 파일과 비교합니다. 결과에 따라 필요한 경우 추가 결정을 내릴지 묻는 메시지가 표시될 수 있으며, Advanced eDiscovery는 누적된 관련성 정보를 기반으로 권장 사항을 제공합니다. 
  
롤링 로드 및 추가 기능은 다음과 같이 다릅니다. 
  
- 일괄 계산 후 새 파일 로드를 가져올 때 Advanced eDiscovery는 파일이 다음 범주 중 하나에 속하는 정도를 결정합니다.
    
  - 유사(동종): 새로운 사용자 지정관련 교육이 필요하지 않습니다. 이전 부하에서 누적된 지식은 새 부하에 "있는 것"으로 적용할 수 있습니다. 
    
  - 고유(이기종): 새로운 사용자 지정관련성 교육이 필요하며 이전 부하에서 누적된 지식은 적용할 수 없습니다. 
    
    이러한 용어는 로드 간 파일 유사성 수준을 참조하며 로드 내에 있지 않습니다. 
    
- 일괄 처리 계산 전과 같은 기간 동안 새 파일 로드를 가져올 때 추가 기능을 사용하면 연합 파일 집합에 대한 업데이트 교육을 계속할 수 있습니다. Advanced eDiscovery는 새 부하가 이전 부하와 유사하거나 다른지 여부를 예측하지 않습니다. 새 부하에 대한 정보를 수집하기만 하면 관련성 교육을 새 파일 및 이전 파일 집합에서 계속 진행할 수 있습니다. 
    
- 관련성 학습에 여러 문제와 일괄 계산 후 문제가 있는 경우 모든 문제에 대해 Catch-up 프로세스가 한 번 수행되고 각 문제에 대해 결과가 계산되고 표시됩니다.
    
> [!NOTE]
> Catch-up 샘플의 크기는 다를 수 있습니다. 이전 부하를 상대로 새 부하의 크기와 새 부하를 추가하기 전에 완료된 샘플 수에 따라 달라 니다. Catch-up 샘플은 일반적으로 새 로드에서 200~2,000개 파일 집합입니다. 
  
> [!TIP]
> Catch-up stops any other tasks and requires individual file tagging and review. 따라서 큰 일괄 처리에 새 파일을 추가할 때 오버헤드를 줄일 수 있습니다. 
  
## <a name="adding-a-new-file-load-using-catch-up-and-rolling-loads"></a>Up 및 Rolling 로드를 사용하여 새 파일 로드 추가

1. In the **Relevance \> Relevance setup** tab, select **Loads.**
    
2. Loads **관리에서** 아이콘을 클릭하여 **+** 로드를 추가합니다. 확인 메시지가 표시됩니다. 
    
3. 계속하려면 **예** 를 클릭합니다. 새 **로드 추가** 대화 상자가 표시됩니다. 
    
    > [!NOTE]
    > 이전 부하에 작업이 수행된 경우 새 부하만 추가할 수 있습니다. 
  
4. 새 로드 추가 **대화** 상자에서  로드  이름 및 설명에 정보를 입력하고 확인을 **클릭합니다.** Advanced eDiscovery는 새 부하를 추가합니다.
    
5. 새 로드 파일을 가져오려면 파일 **추가를 클릭합니다.** 모든 새 파일이 이 로드에 추가됩니다. Advanced eDiscovery가 파일을 가져온 후 롤링 로드 시나리오를 인식하고 추가 기능을 다음 단계로 나타냅니다.
    
6. 대화 **상자 아래쪽에서** 추가를 클릭하여 시나리오를 실행합니다. 
    
    동시 파일 태그 지정을 허용하기 위해 모든 문제에 대해 일반적으로 새 부하에서 200~2,000개 파일을 포함하는 단일 Catch-up 집합이 만들어집니다.
    
    부하가 유사하거나 구분되어 있는지 여부, Advanced eDiscovery가 부하를 자동으로 병합 또는 분할하는지 여부, 그리고 다음 단계의 처리에 대한 정보가 제공됩니다.
    
    그런 다음 파일에 태그를 지정하고 계산 작업을 실행할 수 있습니다. 태그를 지정하면 로드가 유사하거나 고유한지 여부를 확인할 수 있으며 새 파일 집합에서 작업을 계속할 수 있습니다.
    
7. Catch-up set is reviewed, view **Relevance \> Track** for the Catch-up results. 
    
1. 관련성 학습 중에 새 파일 로드가 추가된 경우(즉, 문제가 아직 일괄 계산을 거치지 않은 **경우)** 추가 결과에 관계없이 교육을 계속합니다. 
    
    새 부하와 이전 부하는 하나의 부하로 처리되고, 연합 집합에서 계속된관련성 교육이 진행됩니다. 이제 이 절차를 완료하고 관련성 교육을 계속할 수 있습니다. 
    
2. 새 부하가 일괄 처리 계산 후에 추가된 경우 다음 단계로 진행합니다.
    
8. 일괄 계산 후 추가된 새 부하의 경우 Advanced eDiscovery는 새 부하가 이전 부하와 유사하거나 이전 부하와 유사한지, 아니면 다른지 여부를 다음과 같이 파악합니다.
    
1. 부하가 비슷한 것으로 발견된 경우: 추가적인관련성 교육이 필요하지 않습니다. 대시보드는 **일괄 계산 ** 다시 실행하여 새 부하에 대한 해당성 점수를 계산하는 것이 권장되는 다음 단계를 보여줍니다. 로드는 유사하기 때문에 이전 분류자 분석을 새 파일에서 실행할 수 있습니다. 
    
2. 부하가 고유한 것으로 발견된 경우: 더 많은 타당성 교육이 필요하며 다음 단계는 추가 결정입니다. 다음과 같이 추가 결정을 선택합니다.
    
    부하 **병합을** 선택하면 고급 eDiscovery가 이전 및 새 로드를 교육 집합에 병합합니다. 첫 번째 부하가 일괄 계산을 통과한 것이지만 추가 교육이 필요합니다. 새 부하와 이전 부하를 함께 교육을 계속합니다. 그런 다음 일괄 처리 계산이 다시 실행되고 이전의 일괄 계산 점수는 무시됩니다. 기존 로드에 대한 관련성 점수를 다시 계산할 수 있는 경우(예: 기존 파일 로드 검토가 시작되지 않은 경우) 이 선택을 선택합니다.
    
    분할 **부하를** 선택하는 경우 새 부하에만 대한관련성 교육을 계속합니다. 이 경우 이전 일괄 처리 계산 점수는 그대로 유지됩니다. 기존 부하에 대한 기존 관련성 점수를 다시 계산할 수 없는 경우(예: 기존 부하 검토가 이미 시작된 경우) 이 옵션을 선택합니다. 관련성 점수는 이 시점부터 별도로 관리되고 병합할 수 없습니다.
    
3. 교육 **계속을 클릭합니다.**
